# 大厅真正聚光灯实现

## 实现目标

大厅聚光效果必须来自真正参与 Standard/PBR 逐像素光照、距离衰减、锥角衰减、高光和实时阴影计算的 Cocos `cc.SpotLight`，不能用半透明锥体或 Additive 地面圆盘伪造。

最终节点结构：

```text
Lobby
├─ LobbyOpaqueSurface   builtin-standard
├─ LobbyLampGlow        builtin-unlit / opaque
├─ MainSpotlight        cc.SpotLight / shadow enabled
└─ MainCamera
```

## 已完成的清理

当前实现已经移除：

- 透明锥体光束 Mesh；
- Additive 地面光池 Mesh；
- 玩家正面额外补光灯；
- 角色法线的人工向上偏置；
- 与上述模拟效果配套的 Geometry、顶点着色、材质工厂、运行时调参和测试。

因此，地面亮区、玩家受光、高光和投影只可能来自真实 SpotLight、Cocos Ambient 与 Standard 材质计算，不存在独立假光斑覆盖结果。

## 当前 SpotLight 配置

```ts
export const LOBBY_KEY_LIGHT_CONFIG = Object.freeze({
  nodeName: 'MainSpotlight',
  position: { x: 0, y: 6.65, z: -2 },
  target: { x: 0, y: 0.05, z: -2 },
  up: { x: 0, y: 0, z: 1 },
  color: { red: 255, green: 224, blue: 184 },
  luminousFlux: 8000,
  size: 0.15,
  range: 9,
  spotAngle: 42,
  angleAttenuationStrength: 0.55,
  shadowEnabled: true,
  shadowBias: 0.0001,
  shadowNormalBias: 0.01,
});
```

实际创建逻辑：

```ts
const lightNode = new Node(config.nodeName);
parent.addChild(lightNode);
lightNode.setPosition(config.position.x, config.position.y, config.position.z);
lightNode.lookAt(
  new Vec3(config.target.x, config.target.y, config.target.z),
  new Vec3(config.up.x, config.up.y, config.up.z),
);

const spotLight = lightNode.addComponent(SpotLight);
spotLight.color = new Color(255, 244, 214, 255);
spotLight.size = config.size;
spotLight.luminousFlux = config.luminousFlux;
spotLight.range = config.range;
spotLight.spotAngle = config.spotAngle;
spotLight.angleAttenuationStrength = config.angleAttenuationStrength;
spotLight.shadowEnabled = true;
spotLight.shadowPcf = renderer.scene.PCFType.SOFT_2X;
spotLight.shadowBias = config.shadowBias;
spotLight.shadowNormalBias = config.shadowNormalBias;
```

Cocos Creator 3.8 的运行时代码属性是 `luminousFlux`。Inspector 的显示名称可能使用光通量或 Luminous Power，但代码不使用不存在的 `luminousPower` 字段。

这里必须显式提供 Z 轴 up。灯光目标位于灯的正下方，视线与 `lookAt()` 默认 Y 轴 up 完全共线；Cocos 3.8.8 遇到该退化情况会返回单位旋转，使 SpotLight 保持默认 `-Z` 方向并照向后墙。使用 `(0, 0, 1)` 后，灯的 `-Z` 前向轴才会稳定旋转到世界 `-Y`。

## Standard 材质引用

受光表面必须使用 `builtin-standard`，不能使用 `builtin-unlit`。工程通过场景序列化引用 Editor 创建的材质模板：

```text
assets/resources/lobby/lobby-surface.mtl
└─ EffectAsset = builtin-standard
```

Editor 操作：

1. 在 `assets/resources/lobby` 创建 Material；
2. 命名为 `lobby-surface`；
3. Effect 选择 `builtin-standard`；
4. 将该材质拖到场景 `lobby-entry` 的 `Lobby Surface Material` 字段；
5. 保存场景。

运行时会复制该模板，启用顶点色并关闭全部纹理宏。若字段为空或 Effect 不正确，初始化直接报错。

## 法线与阴影链路

程序几何通过每个三角形的实际绕序计算归一化分面法线，并由 `StaticSurfaceMesh` 上传 Position、Normal、UV、Color 和 Index 流。

```text
程序 Triangle
└─ appendLobbyTriangle()
   ├─ 计算两条边
   ├─ 叉乘得到分面法线
   ├─ 归一化
   └─ 写入三个顶点的 Normal
      └─ Cocos Mesh
         └─ builtin-standard
            ├─ SpotLight 漫反射与高光
            └─ ShadowMap 接收
```

角色法线不做人工偏转。顶部灯照不到的垂直背面保持较暗，这是正常几何光照结果，不再用隐藏补光纠正。

## 关于空气中的光柱

普通 Cocos SpotLight 与 Three.js SpotLight 一样，只计算落在表面上的照明，不自动渲染空气体积散射。

可见空气光柱需要真正的体积雾、散射后处理或其他体积渲染方案。当前需求明确禁止透明锥体模拟，因此本阶段不显示空气光柱，只保留真实表面受光、地面亮区、高光和阴影。

## 验真步骤

开发预览右上角面板已经提供“主射灯启用”开关，可执行完全隔离测试：

1. 把环境光降到 `0`；
2. 开启主射灯，观察玩家、地面亮区和阴影；
3. 关闭主射灯，确认这些结果同时消失；
4. 重新开灯并修改锥角，确认地面受光范围同步变化；
5. 修改范围和光通量，确认照明距离与亮度同步变化；
6. 关闭实时阴影，确认受光仍存在但投影消失。

当前没有透明光柱、地面圆盘或额外补光，所以该测试能直接证明效果来自真正的 SpotLight。
