# 大厅材质与真实聚光灯方案

## 目标画面

大厅保持暗红色工业空间，玩家是唯一视觉焦点：

1. 顶部象牙金日光真实聚光灯照亮玩家和脚下地面；
2. 玩家在地面留下实时阴影；
3. 角色后方红色圆环作为识别轮廓；
4. 墙面、地面和天花板保留较低亮度的暗红环境层次。

暗红环境光用于保证空间可辨认，不能覆盖聚光灯形成的真实明暗边界。

## 最终渲染结构

```text
Lobby
├─ LobbyOpaqueSurface   builtin-standard
├─ LobbyLampGlow        builtin-unlit / opaque
├─ MainSpotlight        cc.SpotLight / shadow enabled
└─ MainCamera
```

本方案不使用：

- 半透明 Geometry 光锥；
- Additive 地面光斑圆盘；
- 玩家正面额外补光灯；
- 自定义 `.effect`；
- 人工扭曲角色法线。

## 在 Cocos Editor 中引用内置 Standard

当前工程不在代码里按名字创建 `builtin-standard`。大厅使用 Editor 创建并序列化引用的 Material 作为模板，代码只复制模板并写入类型化参数。

操作步骤：

1. 在 `assets/resources/lobby` 上右键；
2. 选择“创建 → 材质”；
3. 命名为 `lobby-surface`；
4. 在检查器的 Effect 中选择 Cocos 内置 `builtin-standard`；
5. 选中场景里的 `lobby-entry`；
6. 将 `lobby-surface.mtl` 拖入 `LobbySceneEntry` 的 `Lobby Surface Material` 字段；
7. 保存场景并预览。

工程现有 `assets/resources/lobby/lobby-surface.mtl` 已承担这一职责。若引用丢失或 Effect 不是 `builtin-standard`，运行时会明确拒绝初始化，不做兼容回退。

## Standard 材质参数

```text
USE_VERTEX_COLOR = true
USE_ALBEDO_MAP = false
USE_NORMAL_MAP = false
USE_PBR_MAP = false
USE_OCCLUSION_MAP = false
USE_EMISSIVE_MAP = false
USE_ALPHA_TEST = false

Roughness = 0.62
Metallic = 0
Specular Intensity = 0.46
Emissive = black
```

大厅区域颜色来自不透明 Mesh 的顶点色。Standard 材质负责接收真实 SpotLight、计算分面漫反射与高光，并接收阴影。

## 程序几何与真实法线

程序 Geometry 并不是没有法线。`appendLobbyTriangle()` 按每个三角形的实际绕序计算并归一化分面法线，`StaticSurfaceMesh` 将 Normal 流完整上传给 Cocos Mesh。

当前每个三角形独占三个顶点，因此是明确的 Low Poly 分面法线。角色法线不再增加向上分量，也不为了让顶部灯照亮正面而做人工旋转。

顶点色着色仅产生轻微色差：

| 区域 | RGB |
| --- | ---: |
| 地面 | `(102, 12, 25)` |
| 天花板 | `(48, 5, 14)` |
| 后墙 | `(70, 8, 20)` |
| 前墙 | `(60, 7, 18)` |
| 侧墙 | `(82, 9, 22)` |
| 圆形暗面 | `(30, 3, 10)` |
| 圆形识别环 | `(175, 32, 46)` |
| 角色占位体 | `(190, 86, 78)` |
| 灯线 | `(92, 35, 34)` |
| 灯罩 | `(96, 14, 26)` |

这些颜色不会直接绘制地面聚光区域，地面的亮区必须由 SpotLight 真实计算产生。

## 唯一真实聚光灯

```text
节点名：       MainSpotlight
位置：         (0, 6.65, 0.8)
目标：         (0, 2.83, -2)
颜色：         (255, 244, 214)
光通量：       9100 lm
范围：         9.4
锥角：         45°
边缘衰减：     0.59
光源尺寸：     0.15
阴影：         默认关闭，可调试开启
PCF：          由平台质量配置决定
Shadow Bias：  0.0001
Normal Bias：  0.01
```

灯节点向相机侧前移，并通过 `lookAt(target, up)` 指向玩家躯干中部，避免依靠易错的 Euler 角度。该倾斜方向让角色正面法线获得明显入射分量，使胸部和下肢不再只依赖环境光；同时保留 `(0, 0, 1)` 作为稳定 up。Cocos Creator 3.8 代码使用 `SpotLight.luminousFlux` 设置光通量。

这盏灯负责全部真实局部照明：

- 玩家和地面受光；
- 锥角内外衰减；
- 距离衰减；
- Standard 高光；
- 调试开启后的玩家和大厅几何实时阴影。

`LobbyLampGlow` 只让灯口本身显示为象牙金日光，不参与上述光照计算。

## 暗红环境光

```text
天空环境色： (58, 6, 15)
地面环境色： (22, 1, 6)
Sky Illum：  1580
```

环境光避免大厅未被聚光灯覆盖的区域完全死黑。它与聚光灯职责不同：环境光提供低频暗部基线，SpotLight 提供玩家焦点、亮区边界、高光和阴影。

如果画面仍然太平，应优先降低环境光；如果玩家焦点过暗，再调整主射灯光通量、范围、锥角和相机曝光。

## 吊灯位置

```text
天花板接线点： y = 9.00
灯罩顶部：     y = 7.12
灯罩底部：     y = 6.72
发光面：       y = 6.695
SpotLight：    y = 6.65
```

吊灯保留一段裸露电线，并整体下垂到玩家上方。电线、灯罩和灯口都是实体程序几何，不是光束 Geometry。

## 浏览器灯光调试面板

开发预览右上角提供原生 DOM 调试面板，可实时调整：

- 环境光照度；
- 天空环境色；
- 地面环境色；
- 主射灯光通量；
- 主射灯锥角；
- 主射灯范围；
- 锥角边缘衰减；
- 主射灯启用状态；
- 实时阴影启用状态。

实时阴影初始为关闭状态，必要时可通过面板临时开启。面板只控制 Cocos Ambient 和唯一真实 SpotLight，不控制任何透明光柱、地面光池或玩家补光。参数仅作用于当前预览，不自动写回源码。

## 验证地面亮区来自真实 SpotLight

1. 在调试面板把环境光降到 `0`；
2. 保持主射灯开启，观察玩家、地面亮区和阴影；
3. 关闭主射灯，确认亮区与阴影同时消失；
4. 重新开灯并修改锥角，确认地面受光范围同步变化；
5. 修改范围和光通量，确认距离衰减与亮度同步变化。

当前运行时没有独立地面圆盘，因此不存在“真实灯关闭后假光斑还留在地上”的情况。

## 排查顺序

真实 SpotLight 没有正确照亮表面时，依次检查：

1. `LobbySceneEntry.lobbySurfaceMaterial` 是否引用 `builtin-standard` 材质；
2. `LobbyOpaqueSurface` 是否上传了正确且归一化的 Normal 流；
3. 地面法线是否朝上，墙面是否朝房间内侧；
4. `MainSpotlight` 是否位于角色正前上方并通过 `lookAt(target, Z-up)` 指向玩家躯干；
5. `range`、`spotAngle` 与 `luminousFlux` 是否覆盖目标区域；
6. 相机 HDR 曝光是否把室内场景压得过暗；
7. Ambient 是否过强，导致聚光边界不明显。

不要再通过增加透明 Alpha、叠加 Additive 圆盘、增加隐藏补光或扭曲法线来掩盖真实灯光问题。
