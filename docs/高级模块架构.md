# 战斗行为模块与链式反应系统需求文档

## 1. 项目目标

在现有战场系统中加入一套高度抽象的行为模块框架，使玩家可以在同一局内获得多个模块，并让不同模块、怪物状态和战斗行为组合出新的效果。

系统需要支持：

```text
A 模块
+
B 模块
+
目标当前状态
+
发生的战斗事件
=
新的组合效果
```

示例：

```text
抓取
+ 投掷
= 将怪物作为投射物使用

投掷
+ 怪物中毒
+ 撞击地面
= 在落点产生毒素扩散

投掷
+ 怪物燃烧
+ 撞击其他怪物
= 将燃烧状态传播给其他怪物
```

第一版只实现：

* 抓取模块；
* 投掷模块；
* 对应的技能选择界面；
* 抓取和投掷预览；
* 支持后续组合效果的 ECS 基础架构。

---

# 2. 核心设计原则

## 2.1 模块提供行为，不提供单纯数值

模块主要增加新的战斗动词或改变行为结果，例如：

* 抓取；
* 投掷；
* 牵引；
* 弹反；
* 冲撞；
* 冻结；
* 点燃；
* 碰撞爆炸。

第一版不设计以下模块：

* 伤害增加 20%；
* 生命增加 30%；
* 射速增加 15%。

模块的核心价值是产生新的交互方式，而不是制造数值膨胀。

## 2.2 模块之间不得直接硬编码依赖

禁止在抓取模块中直接编写：

```text
如果玩家同时拥有投掷模块，就启用投掷。
```

抓取模块只负责产生标准状态和事件：

```text
目标进入 Carried 状态
玩家进入 Carrying 状态
触发 EntityGrabbed 事件
```

投掷模块只检查当前是否存在可投掷对象：

```text
玩家处于 Carrying 状态
目标具有 Throwable 能力
```

这样未来加入其他模块时，不需要修改已有模块。

## 2.3 支持任意长度的链式反应

系统应允许事件继续触发新的事件：

```text
怪物被投掷
→ 撞击另一只怪物
→ 被撞怪物击退
→ 撞到场景障碍
→ 触发重击
→ 身上的毒素爆炸
→ 毒素命中附近怪物
```

这里的“无限链式”指架构上不限制固定组合数量，并不允许真正无限循环。

每帧必须设置：

* 最大事件处理数量；
* 单条事件链最大深度；
* 同一实体对同类事件的重复触发限制；
* 循环检测或事件来源标识。

超过安全限制时，应停止后续链式处理并记录诊断信息，避免死循环和卡帧。

# 3. ECS 架构需求

## 3.1 实体状态组件

第一版至少需要以下组件或等价 SoA 字段。

### Grabbable

表示实体是否允许被抓取。

包含：

* 是否可抓取；
* 当前生命比例要求；
* 体型等级；
* 抓取抗性；
* 是否允许被玩家抓取。

第一版规则：

* 仅允许抓取普通小型怪物；
* 怪物生命值低于 50% 后进入可抓取状态；
* 精英怪和大型怪默认不可抓取。

### Carried

表示实体当前正被其他实体携带。

包含：

* 携带者实体 ID；
* 携带位置偏移；
* 当前携带时间；
* 是否允许投掷。

### Carrying

表示玩家当前正在携带一个实体。

包含：

* 被携带实体 ID；
* 抓取动作阶段；
* 当前技能上下文。

### Throwable

表示实体可以被投掷。

包含：

* 基础投掷质量；
* 最大投掷距离；
* 碰撞半径；
* 撞击强度系数。

### Thrown

表示实体正在作为投射物飞行。

包含：

* 起始位置；
* 当前速度；
* 投掷方向；
* 投掷者实体 ID；
* 已飞行距离；
* 最大飞行距离；
* 当前命中历史。

### CombatTags

提供通用状态标签，例如：

* Poisoned；
* Burning；
* Frozen；
* Explosive；
* Armored；
* Elite；
* SmallBody；
* Executable。

标签必须由统一注册表管理，禁止各模块自行使用任意字符串。

---

## 3.2 标准战斗事件

所有模块通过标准事件交换信息。

第一版至少支持：

```text
EntityBecameGrabbable
GrabStarted
EntityGrabbed
GrabCancelled
ThrowAimingStarted
EntityThrown
EntityCollision
GroundImpact
EntityImpact
HeavyImpact
EntityKilled
StatusApplied
StatusRemoved
```

每个事件至少包含：

* 事件类型；
* 来源实体；
* 目标实体；
* 技能或模块来源；
* 事件链 ID；
* 当前链式深度；
* 世界位置；
* 方向；
* 强度；
* 标签快照。

事件应写入预分配的固定容量缓冲区，避免战斗热路径产生大量临时对象。

---

## 3.3 模块结构

所有模块使用统一定义。

模块定义至少包含：

```text
模块 ID
显示名称
图标
模块类别
触发方式
前置条件
产生的行为
冷却规则
预览类型
输入模式
事件监听列表
```

模块类别包括：

### ActiveAction

需要玩家主动操作，例如抓取、投掷、牵引。

### PassiveCondition

被动添加条件，例如怪物半血后进入可处决状态。

### ActionModifier

修改现有动作，例如投掷距离增加、投掷目标可以穿透。

### Reaction

监听事件和标签组合，并产生新的效果。

第一版的抓取与投掷属于 `ActiveAction`。

---

## 3.4 系统执行阶段

推荐按照以下阶段运行：

```text
Input
→ SkillSelection
→ SkillPreview
→ ActionValidation
→ ActionExecution
→ MovementAndThrownSimulation
→ Collision
→ EventResolution
→ ReactionResolution
→ DamageAndLifecycle
→ RenderSynchronization
```

模块不能直接跨阶段修改所有状态。

例如：

* 输入阶段只产生技能意图；
* 抓取系统在 ActionExecution 中写入携带状态；
* 投掷系统在 ActionExecution 中写入 Thrown 状态；
* 碰撞系统只产生撞击事件；
* 反应系统根据事件和标签产生组合效果；
* 伤害系统最后统一处理伤害和斩杀。

# 4. 第一版模块

## 4.1 抓取模块

### 触发条件

玩家满足以下条件时可以抓取：

* 当前没有携带其他实体；
* 目标具有 `Grabbable`；
* 目标具有 `Executable`；
* 目标位于抓取距离内；
* 玩家和目标之间不存在明显障碍；
* 抓取技能不处于冷却状态。

第一版中，当普通小型怪物生命值低于 50% 时，由状态系统自动添加 `Executable`。

### 操作方式

玩家选择抓取技能后：

1. 按住抓取技能按钮；
2. 显示蓝色方向箭头；
3. 系统搜索箭头方向内最近的合法目标；
4. 合法目标头顶显示抓取标志；
5. 松开按钮执行抓取；
6. 没有合法目标时不执行技能；
7. 拖出取消区域或按下闪避可以取消预览。

### 执行效果

抓取成功后：

* 目标停止普通移动和攻击；
* 目标绑定到玩家前上方的携带位置；
* 目标进入 `Carried`；
* 玩家进入 `Carrying`；
* 触发 `EntityGrabbed`；
* 技能上下文自动切换为投掷。

抓取本身不直接斩杀目标。

斩杀由后续重摔、投掷撞击或其他反应模块决定。

---

## 4.2 投掷模块

### 触发条件

投掷技能只有在玩家正在携带实体时可用。

### 操作方式

1. 玩家按住投掷技能按钮；
2. 按钮进入虚拟摇杆模式；
3. 从按钮中心拖动确定投掷方向；
4. 拖动距离决定投掷距离；
5. 场景中显示投掷落点圆圈；
6. 显示从玩家到落点的轨迹预览；
7. 松开按钮执行投掷；
8. 拖回中心或拖入取消区域取消投掷。

### 投掷距离

投掷距离由输入幅度决定：

```text
最小有效拖动
→ 近距离投掷

最大拖动
→ 最大距离投掷
```

第一版不要求模拟复杂抛物线，可使用低弧线或平面投掷。

落点预览必须考虑：

* 最大距离；
* 场景障碍；
* 地面位置；
* 被投掷怪物碰撞体；
* 不可到达区域。

### 撞击结果

被投掷怪物撞击地面或其他怪物时：

* 触发 `GroundImpact` 或 `EntityImpact`；
* 根据投掷速度、质量和飞行距离计算撞击强度；
* 如果原怪物仍处于可处决状态，重型撞击可以直接斩杀；
* 被撞击的其他怪物受到伤害和击退；
* 后续反应模块可以监听该事件产生组合效果。

# 5. 技能选择界面

## 5.1 界面位置

在右侧瞄准摇杆上方增加技能操作区域。

区域由以下部分组成：

* 一个当前选中技能按钮；
* 左侧一个上一个技能图标；
* 右侧一个下一个技能图标。

屏幕中同时最多显示三个技能图标。

中间图标尺寸最大，表示当前技能。

左右图标尺寸较小并降低透明度。

## 5.2 轮盘选择

技能列表以循环方式排列：

```text
抓取
→ 投掷
→ 其他技能
→ 抓取
```

玩家可以：

* 在技能区域左右滑动；
* 沿弧形方向拖动；
* 点击左右的小图标；
* 使用桌面调试按键切换。

切换时图标沿圆弧旋转，形成类似俄罗斯轮盘的循环选择效果。

每次只能有一个当前选中技能。

## 5.3 上下文技能切换

抓取成功后，系统可以自动将投掷技能移动到中间位置。

投掷完成或取消携带后，恢复玩家上一次手动选中的技能。

无效技能需要：

* 保留在轮盘中；
* 显示为不可用状态；
* 降低亮度；
* 不响应执行；
* 显示简短原因，例如“需要先抓取目标”。

第一版只有抓取和投掷时，可以保留一个空的预留技能位，用于验证三图标布局。

# 6. 技能预览效果

## 6.1 抓取预览

抓取按钮按住后显示：

### 蓝色方向箭头

箭头从玩家位置向瞄准方向延伸。

箭头需要表示：

* 抓取方向；
* 最大抓取距离；
* 当前是否存在合法目标。

没有合法目标时使用较暗的蓝色。

存在合法目标时箭头增强亮度。

### 目标头顶标志

合法抓取目标头顶显示专用标志。

标志需要：

* 跟随目标世界位置；
* 始终朝向摄像机；
* 遮挡时可以降低透明度；
* 目标失效后立即隐藏。

目标选择优先级：

1. 位于方向箭头范围内；
2. 距离玩家较近；
3. 与箭头方向夹角较小；
4. 当前仍然可抓取。

---

## 6.2 投掷预览

玩家携带怪物并按住投掷技能后显示：

### 投掷方向

根据技能按钮摇杆输入显示方向线。

### 落点圆圈

地面显示一个圆形落点范围。

圆圈表示：

* 当前预计落点；
* 撞击影响范围；
* 是否被障碍物阻挡。

有效落点使用正常显示。

无效落点使用警告状态，并限制到最后一个合法位置。

### 投掷距离控制

按钮拖动幅度映射为投掷距离。

```text
输入幅度 0
→ 不投掷

输入幅度接近 Dead Zone
→ 最短投掷

输入幅度 1
→ 最大投掷距离
```

释放按钮后，按照最终方向和距离执行投掷。

# 7. MVP 范围

第一版只实现：

* 通用模块注册表；
* 标准战斗事件缓冲；
* 事件链 ID 和深度限制；
* Grabbable、Carried、Carrying、Throwable、Thrown；
* 抓取模块；
* 投掷模块；
* 半血普通怪进入可抓取状态；
* 投掷撞击地面；
* 投掷撞击其他怪物；
* 重型撞击斩杀被投掷怪物；
* 右摇杆上方技能轮盘；
* 同时显示三个技能图标；
* 抓取方向箭头；
* 抓取目标头顶标志；
* 投掷方向和落点圆圈；
* 投掷距离摇杆控制；
* 对应的调试面板和自动化测试。

第一版不实现：

* 中毒、燃烧、冰冻的组合反应；
* 模块等级；
* 模块碎片；
* 模块品质；
* 多人同步；
* 复杂抛物线物理；
* 大型怪物抓取；
* 精英怪抓取；
* 超过一个被携带目标。

# 8. 验收标准

## 抓取

* 普通小型怪物生命低于 50% 后出现可抓取标志；
* 玩家按住抓取按钮时出现蓝色方向箭头；
* 合法目标头顶出现唯一目标标志；
* 松开按钮后目标被抓到玩家前上方；
* 不合法目标不能被抓取；
* 抓取过程中怪物不再独立移动和攻击。

## 投掷

* 抓取成功后投掷技能可用；
* 按住投掷按钮后进入摇杆输入；
* 拖动方向改变投掷方向；
* 拖动幅度改变投掷距离；
* 场景中实时显示落点圆圈；
* 松开后怪物按照预览方向飞出；
* 撞击地面或怪物时产生标准撞击事件；
* 满足斩杀条件时，被投掷怪物立即死亡。

## 技能轮盘

* 位于右侧瞄准摇杆上方；
* 同时只显示三个图标；
* 中间图标表示当前选中技能；
* 技能可以循环选择；
* 抓取成功后可以快速切换至投掷；
* 不可用技能具有明确禁用表现。

## ECS 组合能力

* 抓取模块不直接引用投掷模块；
* 投掷模块不直接引用具体怪物类型；
* 撞击系统不直接编写未来的毒爆或燃烧逻辑；
* 新增 Reaction 模块时，不需要修改抓取和投掷实现；
* 事件链超过安全限制时能被终止并记录；
* 战斗热路径不产生不受控的逐帧对象分配。
