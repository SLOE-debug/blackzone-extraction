## Venom Lobber：最终改成“毒茧破土 → 跪伏塌缩”

### 为什么现在出生仍然怪

当前出生过程仍然把完整身体作为一个 Root，从地下 `-7.45` 移动到地面，同时做前后偏移和最多约 10° 的 Pitch。

六条腿又在另一个系统里，根据同一时间重新计算 Root 高度、前向偏移和 Pitch，再单独求 IK。

渲染时则依次叠加：

```text
Root 上升
Root 前移
全身 Pitch
腿部 IK
尾部旋转
Body Bob
```

这很容易形成一种“完整模型从地下滑出来，腿再努力追赶身体”的感觉。

### 敲定的出生演出：毒茧破土

总时长设为 **1.45 秒**。

#### 0.00–0.22 秒：地面隆起

* 只显示一个低多边形暗绿色土包。
* 本体完全隐藏。
* 土包上出现三条短裂纹。
* 不显示大面积酸池。
* 无碰撞、不可受击。

#### 0.22–0.52 秒：毒茧开裂

* 土包裂成 6 块低多边形甲壳碎片。
* 尾刺尖端先露出，但不负责牵引整个身体。
* 毒囊产生一次低亮度脉冲。
* 本体保持直立压缩姿态，不允许全局 Pitch。

#### 0.52–1.10 秒：身体展开

本体从一个紧缩的蹲伏姿态展开：

```text
前腿落地
→ 中腿落地
→ 后腿落地
```

每组间隔 `0.1 秒`。

Root 只做垂直上升：

```text
rootForward = 0
rootPitch = 0
```

身体不会横着、斜着或沿曲线滑出。

#### 1.10–1.45 秒：站稳

* 身体向下压 `0.12` 个局部单位。
* 毒囊从 70% 膨胀到正常体积。
* 尾刺展开至待机位置。
* 毒茧碎片缩小并沉入地下。
* 进入 Alive 后继续保留当前 `0.85 秒` 攻击锁定。

### 代码结构

不要再让 Animation System、Leg System 和 Renderer 分别根据 `stateTime` 推导出生曲线。

新增统一的生命周期姿态系统：

```ts
class VenomLobberLifecyclePoseSystem {
  public update(state: VenomLobberState, deltaTime: number): void;
}
```

由它一次性写入：

```ts
animation.rootElevation[index];
animation.bodyCompression[index];
animation.venomSacScale[index];
animation.tailCurl[index];
animation.cocoonOpen[index];
animation.lifecycleLegProgress[index];
```

腿部系统只读取 `lifecycleLegProgress`，Renderer 只消费最终姿态，不再自行解释 `stateTime`。

## 死亡方式：取消侧翻和飞溅，改为跪伏塌缩

当前死亡同时发生：

* 全身侧翻约 35°

* 六足向身体折叠

* 毒囊内爆

* 尾部摆动

* 1.08 秒后整体下沉

同时，进入 `Dying` 的第一帧就创建死亡效果，但液滴和残迹要到 `0.68 秒`才真正展开。

这些动作缺少共同重心，所以看起来像几套特效叠在一起。

### 敲定的死亡演出：毒囊失压

总时长设为 **1.55 秒**。

#### 0.00–0.18 秒：受创僵直

* 停止移动和攻击。
* 尾刺迅速失去张力。
* 身体向后轻微震动。
* 不侧翻。

#### 0.18–0.62 秒：六足失力

依次折叠：

```text
前腿
→ 中腿
→ 后腿
```

身体重心垂直下降，前甲壳先接近地面。

不再使用：

```ts
deathRoll
```

整个身体始终保持基本朝上。

#### 0.62–1.05 秒：毒囊泄压

* 毒囊尺寸从 100% 缩至 25%。
* 亮绿色逐渐变成深灰绿色。
* 尾部绕向身体侧面。
* 身体甲壳发生轻微纵向压缩。
* 不生成五枚四面体飞溅物。

#### 1.05–1.55 秒：残躯塌陷

* 身体高度压缩到 35%。
* 整体下沉约 `0.35` 个局部单位。
* 在身体脚印范围内生成一块小型暗色残迹。
* 残迹额外保留 `1 秒`，从边缘向中心收缩。

死亡残迹应在 `stateTime >= 1.05` 时才生成，而不是进入 Dying 就立即创建。

需要删除：

```text
deathRoll
五枚装饰飞溅液滴
大范围死亡液面
```

最终视觉语言会变成：

```text
出生：毒茧在地下形成，怪物直立展开
战斗：毒囊充能并发射毒液
死亡：毒囊失压，六足跪伏，甲壳塌回地面
```

# 玩家操作：彻底取消左摇杆自动瞄准和自动攻击

当前右摇杆未激活时，控制器会把**左摇杆移动方向**复制成瞄准方向：

```ts
if (movementLength > DIRECTION_EPSILON) {
  aimDirection = movementDirection;
}
```

随后调用 `resolveAutoTarget()`，找到目标后自动转向，并在角色朝向对齐后自动开火。

这条链路必须完整删除。

## 最终控制方案：标准双摇杆射击

### 左摇杆

只负责：

```text
移动
```

左摇杆绝不再：

* 搜索敌人
* 改变瞄准方向
* 锁定敌人
* 触发射击
* 影响枪口方向

### 右摇杆

同时负责：

```text
瞄准方向 + 射击意图
```

规则直接定为：

```text
右摇杆超过 Dead Zone
→ 角色朝右摇杆方向瞄准
→ 立即开始射击

持续按住
→ 根据武器射速持续射击

松开右摇杆
→ 立即停止射击
```

所有武器使用同一套操作：

* 手枪受自身射击间隔限制。
* 霰弹枪受泵动间隔限制。
* 步枪和冲锋枪连续开火。
* 不需要额外射击按钮。

右摇杆当前已经提供归一化瞄准方向和 `aiming` 状态，因此输入基础已经存在。

---

## 只在右摇杆激活时启用轻度辅助瞄准

辅助瞄准必须满足：

```text
右摇杆正在输入
+
目标位于右摇杆方向附近
```

敲定参数：

```text
最大辅助角度：6°
最大辅助距离：16
方向修正比例：最多 30%
```

不要把方向完全吸到敌人中心。采用方向混合：

```ts
finalAimX = normalize(
  manualAimX * 0.7 + assistedAimX * 0.3,
);

finalAimZ = normalize(
  manualAimZ * 0.7 + assistedAimZ * 0.3,
);
```

玩家始终控制主方向，系统只做轻微校正。

没有目标时照样朝右摇杆方向开火，不能要求 `targetLocked` 才允许射击。当前 `shouldFireAtLockedTarget()` 明确要求目标存在，因此应删除。

无目标时写一个远处自由射击点：

```ts
const FREE_AIM_DISTANCE = 32;

fireTarget.x = player.positionX + aimDirection.x * FREE_AIM_DISTANCE;
fireTarget.y = player.positionY + PLAYER_WEAPON_AIM_HEIGHT;
fireTarget.z = player.positionZ + aimDirection.z * FREE_AIM_DISTANCE;
```

---

## 控制器应该改成这样

核心逻辑：

```ts
if (!controls.aiming || !weaponEquipped) {
  intent.aiming = false;
  player.setControlIntent(intent);
  return false;
}

cameraRig.writeWorldPlanarDirection(
  controls.aimX,
  controls.aimY,
  this.aimDirection,
);

const assisted = monsters.resolveAimTarget(
  player.positionX,
  player.positionZ,
  this.aimDirection.x,
  this.aimDirection.z,
  this.aimTarget,
);

if (assisted) {
  this.applySoftAimCorrection(
    player.positionX,
    player.positionZ,
  );
  this.copyLockedTarget(fireTarget);
} else {
  this.writeFreeAimTarget(player, fireTarget);
}

intent.aimX = this.aimDirection.x;
intent.aimZ = this.aimDirection.z;
intent.aiming = true;

player.setControlIntent(intent);

return true;
```

明确删除：

```ts
writeAutomaticLockDirection()
resolveAutoTarget()
shouldFireAtLockedTarget()
```

这样左摇杆移动时，怪物查询都不会执行，也不会产生任何自动射击。

## 场景交互按钮独立出来

目前接近宝箱或装备时，系统会把右摇杆切换成场景操作按钮，并直接关闭瞄准。

这也应该一起改掉。

最终布局：

```text
左下：移动摇杆
右下：瞄准与射击摇杆
右摇杆上方：临时交互按钮
```

交互按钮只在附近存在可交互对象时出现：

* 开箱
* 捡装备
* 其他场景操作

右摇杆永远保持射击职责，不因靠近宝箱而突然失效。

## 最终操作定义

```text
左摇杆移动
右摇杆瞄准并射击
松开右摇杆停止射击
右摇杆输入期间提供轻度方向修正
交互使用独立按钮
空白屏幕拖动旋转相机
```

这套操作不会再出现“只是往左跑，角色却自动锁定并射击”的行为，同时保留移动端双摇杆射击所需的响应速度。
