## 结论

这三个问题里：

* **交互按钮位置是确定的 UI 回归**：代码新建了独立 `interactionButton`，并明确放在右摇杆上方。
* **当前瞄准逻辑不符合新需求**：目标方向会以 30% 权重修改玩家的水平射击方向，所以仍然属于辅助瞄准。
* **子弹伤害链路表面上已经接通，但存在时序、纵向轨迹和测试覆盖三个明显风险点**。需要先加观测数据，避免继续盲修。

## 1. 右摇杆复用方案

当前 `VirtualJoystick` 在设置 Action Icon 后，会从摇杆模式直接切换成按钮模式：触摸只产生 `actionPressPending`，不再输出摇杆方向。

不建议只是简单恢复旧逻辑，因为玩家靠近箱子后，右摇杆会完全不能射击。

### 推荐交互

右摇杆保持原位置，操作图标显示在摇杆中心：

* **点击并在死区内松开**：执行开箱、拾取。
* **拖出死区**：取消本次交互候选，立即进入瞄准和射击。
* 拖动后松手不会误触交互。
* 场景没有可交互对象时，不显示中心图标，行为与普通右摇杆完全一致。

### 修改范围

`battlefield-control-hud.ts`

* 删除 `interactionButton` 字段、创建、布局和销毁逻辑。
* `setContextAction()` 改为设置 `aimJoystick` 的 Action Icon。
* `consumeContextActionPress()` 重新消费右摇杆事件。
* `BattlefieldGameplayGraphics.synchronize()` 恢复只接收两个摇杆。

`battlefield-control-style.ts`

* 删除 `interaction` 样式和 `interactionVerticalGap`。当前独立按钮尺寸就是从这里引入的。

`virtual-joystick.ts`

* 将 Action Icon 从“模式开关”改成“视觉叠加＋点击候选”。
* 增加类似 `actionGestureCandidate` 的状态。
* TOUCH_MOVE 超过死区后取消 Action，正常写入轴值。
* TOUCH_END 时仍位于死区，才产生一次 Action Press。

## 2. 瞄准系统方案

需求应拆成两个完全独立的维度：

* **XZ 水平方向：100% 来自右摇杆，不允许修正。**
* **Y 轴俯仰：只有右摇杆方向附近存在怪物时才修正。**

当前实现会调用 `applySoftAimCorrection()`，直接改变 `aimDirection.x/z`，必须删除。

### 新的射击规则

没有候选怪物：

```text
水平射击方向 = 右摇杆世界方向
射击高度 = 当前真实枪口高度
pitch = 0
```

方向内存在怪物：

```text
水平射击方向 = 右摇杆世界方向，保持不变
射击高度 = 候选怪物有效命中高度
只计算 pitch
```

重点是：**怪物只能提供 Y，高度不能提供 XZ。**

### 数据结构建议

不要再让 Aim Controller 输出一个完整世界目标点，改成：

```ts
interface BattlefieldFireIntent {
  directionX: number;
  directionZ: number;
  targetElevation: number | null;
}
```

在枪口姿态已经确定后，由武器系统生成三维轨迹：

```ts
const targetDistance = candidate
  ? projectedMonsterDistance
  : projectile.maximumRange;

targetX = muzzleX + directionX * targetDistance;
targetZ = muzzleZ + directionZ * targetDistance;
targetY = targetElevation ?? muzzleY;
```

这样空射时能真正与枪口齐平。

当前自由射击高度写成了 `player.positionY + 2.35`，但实际弹丸方向是从 `muzzle.muzzleY` 指向这个点，两者不相等时，子弹会产生额外上仰或下压。

需要删除：

* `correctionWeight`
* `writeSoftAimDirection()`
* `applySoftAimCorrection()`

保留：

* 最大检测角度。
* 最大检测距离。
* 候选怪物高度查询。

## 3. 子弹碰撞与不掉血问题

当前链路是：

```text
integrate
→ findFirstProjectileHit
→ ImpactBuffer
→ damageMonster
```

碰撞和伤害调用确实存在。

但有三个需要优先处理的问题。

### 问题 A：新子弹在碰撞之后才生成

现在两个系统都在 Combat 阶段：

* 弹丸碰撞 `order = 0`
* 武器生成弹丸 `order = 10`

也就是说，本帧按下射击后，新子弹要到下一帧才能第一次积分和碰撞。

这未必是“永远不造成伤害”的唯一根因，但这是明确的帧时序错位。

建议顺序调整为：

```text
Input / Aim
→ Weapon Spawn
→ Monster Simulation
→ Projectile Integration
→ Spatial Index
→ Projectile Collision
→ Damage Resolution
→ Rendering
```

把 `BattlefieldWeaponWorldSystem` 移到 `PreSimulation`，或者放在 `Simulation` 且排在 Projectile Integration 前面。

### 问题 B：3D 碰撞与射击高度不一致

子弹使用完整三维扫掠，而小型怪物的碰撞体高度远低于玩家持枪高度。空射目标目前使用固定玩家高度，而不是枪口高度；当没有识别到辅助目标时，子弹很容易从 Curve Crawler 身体上方飞过。

这会在俯视画面中看起来像“子弹穿过怪物”，但实际上是 Y 轴没有相交。

完成第二部分的瞄准重构后，这一问题会同时被消除：

* 方向范围内有怪：只对准怪物高度。
* 没有怪：严格水平射击。

### 问题 C：测试没有覆盖完整伤害链路

现有测试只验证了：

* Swept Sphere 几何函数。
* Impact Buffer 是否保存数据。

没有验证：

```text
真实 Projectile Population
→ Crowd 宽相位
→ 怪物窄相位
→ population/entity 路由
→ 怪物 health 下降
```

因此底层数学测试全绿，也不能证明游戏内会掉血。

另外，Curve Crawler 当前碰撞体明确只包含胸腔和腹部，不包含腿部。子弹擦过腿部不会造成伤害，这是设计行为，但需要用 Debug 碰撞体显示出来，否则玩家会认为是穿模。

## 落地顺序

### 第一批：先恢复操作体验

1. 删除右摇杆上方独立按钮。
2. 右摇杆实现“点击交互、拖动射击”。
3. 补充点击、拖动、取消触摸、多点触控测试。

### 第二批：重做射击意图

1. XZ 永远使用手动方向。
2. 怪物查询只输出目标高度。
3. 空射使用真实枪口 Y。
4. 删除 30% 水平修正。

### 第三批：修复并验证碰撞链路

先增加临时统计：

```text
projectilesSpawned
projectilesIntegrated
broadPhaseCandidates
narrowPhaseHits
impactsQueued
damageEventsApplied
```

根据数据可以立即定位：

* `candidates = 0`：Crowd 索引或坐标转换问题。
* `candidates > 0，hits = 0`：Y 轴或复合碰撞体问题。
* `hits > 0，impacts = 0`：Projectile Population 穿透循环问题。
* `impacts > 0，damage = 0`：populationId/entityId 路由问题。

最后补完整回归场景：

* 静止 Curve Crawler 正面命中。
* 横向移动 Curve Crawler 的 CCD 命中。
* Venom Lobber 命中。
* 30、60、120 FPS 下结果一致。
* 两只怪物贯穿及伤害衰减。
* 方向内有怪时，XZ 完全不变化，只改变 Y。
* 方向外有怪时，子弹保持枪口水平高度。
