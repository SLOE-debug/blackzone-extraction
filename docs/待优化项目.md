## 1. 怪物防堆叠目前确实是蜘蛛专用

当前 `CurveCrawlerSeparationSystem`：

* 直接依赖 `CurveCrawlerState`
* 直接读取蜘蛛 morphology
* 使用蜘蛛专属的 separation profile
* 只在同一个蜘蛛群体内部建立空间哈希

Venom Lobber 的更新顺序只有生命、战斗、移动、动画和渲染，没有 Separation。

而通用 `MonsterPopulation` 目前只提供 `count / update / dispatch`，没有位置、碰撞半径、重量和 Crowd 注册能力。

### 我敲定的重构方案

把蜘蛛自己的 Separation 删除，改成 **Battlefield World 级别的统一 Crowd Constraint**：

```text
所有怪物移动
    ↓
统一构建 Monster Spatial Index
    ↓
跨种类 Separation
    ↓
Projectile / Aim / AOE 共用同一索引
    ↓
渲染同步
```

核心抽象：

```ts
export interface PlanarCrowdPopulation {
  readonly populationId: number;
  readonly count: number;

  readonly lifecycle: Uint8Array;
  readonly x: Float32Array;
  readonly y: Float32Array;
  readonly radius: Float32Array;
  readonly inverseMass: Float32Array;
}
```

通用系统：

```ts
export class PlanarCrowdSeparationSystem {
  public register(population: PlanarCrowdPopulation): void;
  public unregister(populationId: number): void;
  public rebuild(): void;
  public solve(deltaTime: number): void;
}
```

权重直接这样定：

```text
Curve Crawler inverseMass = 1.0
Venom Lobber inverseMass  = 0.28
```

所以蜘蛛撞到 Venom Lobber 时，主要由蜘蛛让路；两个 Venom Lobber 相遇时才共同修正。

这个系统应该放到：

```text
assets/core/monsters/crowd/
```

战场层只负责注册 Population。以后新增任何怪物，只需要提供平面位置、半径和重量，不再复制一份空间哈希。

## 2. 毒池“大风车”的原因已经确定

截图中的风车感来自这段逻辑：

```ts
const even = ray % 2 === 0;

const red = catalyzed
  ? even ? 0.62 : 0.22
  : even ? 0.18 : 0.08;
```

当前毒池是一个 14 射线 Triangle Fan，每个扇区交替使用亮色和暗色。所有三角形又共享中心点，所以自然形成非常明显的放射状拼色风车。

### 正式修改方案：不规则腐蚀液面

彻底移除奇偶扇区换色。

毒池固定为三层，但仍然留在同一个 Effect Batch：

```text
第一层：不规则暗色液面
第二层：断裂的内侧毒纹
第三层：少量低多边形气泡
```

具体拓扑：

```ts
const POOL_OUTER_SEGMENTS = 18;
const POOL_VEIN_COUNT = 3;
const POOL_BUBBLE_COUNT = 4;
```

基础液面：

* 中心颜色统一使用深色
* 外圈使用稍亮、但相同色系的颜色
* 每个外圈半径可以继续按 Seed 扰动
* 禁止按三角形交替换色

普通毒池色板：

```text
中心：#12382A
主体：#1C5936
边缘：#327A48
毒纹：#58A866
气泡：#74C778
```

Catalyzed 毒池：

```text
中心：#35451D
主体：#68742A
边缘：#A5A53C
毒纹：#D2C85A
气泡：#E0D86A
```

Catalyzed 也不再用荧光绿和黑绿交替，只改变整个色板。

落地过程定为：

```text
0.00–0.12 秒：中心冲击点快速扩张
0.12–0.32 秒：液面沿不规则轮廓铺开
持续期：毒纹缓慢脉冲，4 个气泡交替隆起
结束前 0.7 秒：整体缩小并逐渐变暗
```

这会保持 Low Poly 风格，但不再像旋转警示牌。

## 3. 远处怪物瞬间消失不是 LOD，是直接回收

蜘蛛超出 `recycleRadius` 后，代码立即执行：

```ts
transitionMonsterLifecycle(
  vitality,
  index,
  MonsterLifecycleState.Dormant,
);
```

Venom Lobber 也是相同逻辑，超出距离后直接进入 `Dormant`。

蜘蛛的可见性系统本身只有视锥测试，没有显式的距离简化逻辑。

所以违和感的真正来源是：

```text
Alive
→ 超过回收半径
→ 同一帧 Dormant
→ 索引从 GPU 批次删除
```

### 正式修改方案：增加 `Despawning`

通用生命周期增加：

```ts
export enum MonsterLifecycleState {
  Dormant,
  Spawning,
  Alive,
  Despawning,
  Dying,
  DeathComplete,
}
```

`Despawning` 仍然属于 Resident。

回收规则定为：

```text
超过软回收半径
+
已经位于视锥外
+
最近 1 秒没有受击
+
没有正在攻击玩家
=
进入 Despawning
```

硬半径用于兜底：

```text
Curve Crawler hardRecycleRadius = 68
Venom Lobber hardRecycleRadius  = 76
```

超过硬半径但仍在镜头内，也不能直接隐藏，而是播放加速版撤离动画。

具体表现：

* Curve Crawler：腿向身体收拢，身体下沉，地裂闭合，持续 `0.45 秒`
* Venom Lobber：尾部先缩回，六足内折，身体沉入小块暗色毒渍，持续 `0.7 秒`

只有撤离演出结束才进入 `Dormant`。

这不是距离 LOD，而是给对象池回收补上一个可见的生命周期过渡。

## 4. 枪械确实需要穿透，而且当前步枪伤害偏低

当前伤害：

| 武器           |      当前伤害 |
| ------------ | --------: |
| Desert Eagle |        32 |
| KRISS Vector |        13 |
| M4A1         |        19 |
| AKM          |        27 |
| Pump Shotgun | 每颗 20，9 颗 |

普通蜘蛛有 100 HP，Venom Lobber 有 240 HP。

现在每颗子弹只调用 `damageFirstAlongSegment()`，命中首个目标后立即停用，所以数据结构本身不支持穿透。

武器定义中也只有速度、射程和命中半径，没有穿透参数。

### 我敲定的武器数值

| 武器           |   新伤害 | 最大命中目标 | 每次穿透保留伤害 |
| ------------ | ----: | -----: | -------: |
| Desert Eagle |    34 |      1 |      不穿透 |
| KRISS Vector |    17 |      2 |      62% |
| M4A1         |    26 |      3 |      72% |
| AKM          |    38 |      4 |      78% |
| Pump Shotgun | 每颗 22 |   每颗 2 |      58% |

伤害计算：

```ts
damageForTarget = baseDamage * Math.pow(
  damageRetention,
  targetOrder,
);
```

例如 AKM：

```text
第 1 只：38
第 2 只：29.6
第 3 只：23.1
第 4 只：18.0
```

这样 AKM 会真正体现“大威力步枪”的清线能力，但不会对整条路径上的怪物造成完整伤害。

### 命中架构直接改成 Hitscan

我不建议继续让高速枪弹承担逐帧碰撞。

正式规则：

```text
手枪、步枪、冲锋枪、霰弹枪：
开火帧完成 Hitscan 与多目标穿透
仍创建无碰撞的视觉 Tracer

毒液弹、手雷、慢速魔法：
继续使用实体 Projectile
```

接口改成：

```ts
damageAlongSegment(
  query: Readonly<PlanarPenetratingHitQuery>,
  hits: MutablePlanarHitBuffer,
): number;
```

`MutablePlanarHitBuffer` 使用固定 TypedArray，收集一次线段内的候选，按 `segmentProgress` 排序并去重。

不要采用：

```text
命中一次
→ 缩短线段
→ 再全量查询一次
```

那会把穿透次数乘进查询成本。应该一次空间索引遍历完成全部候选收集。

## 5. Venom Lobber 的出场和死亡，我敲定这一套

现在的出场主要是从地下向上抬升、腿部展开和轻微扭转；死亡则是侧翻并下沉。

当前总时长是：

```text
出生 1.28 秒
死亡 1.52 秒
```

### 出场：毒囊裂土

总时长改为 `1.6 秒`。

**0.00–0.32 秒：地下预兆**

地面先出现一块小型暗绿色不规则毒渍，三条短毒纹从中心缓慢展开。尾刺尖端先刺出地面，毒囊在地下产生一次微弱亮光。

此时：

```text
无碰撞
不可受击
不可施法
```

**0.32–1.05 秒：尾部牵引身体出土**

尾部先向上弯曲，毒囊随后露出，身体被尾部动作“牵”出地面，而不是整个模型垂直上升。

六条腿按三组依次展开：

```text
后腿
→ 中腿
→ 前腿
```

每组延迟 `0.08 秒`，避免六条腿同时机械展开。

**1.05–1.60 秒：落地稳定**

身体向下压一次，毒囊发生一次明显但柔和的脉冲，地面毒纹收回，只保留一个很小的暗色痕迹。

进入 Alive 后增加：

```text
initialAttackLockSeconds = 0.85
```

防止刚出现就立刻投射毒弹。

---

### 死亡：毒囊塌缩

总时长改为 `1.85 秒`。

**0.00–0.18 秒：死亡冲击**

立即取消施法，尾刺快速回缩，身体短暂停顿。不是全局 Hit Stop，只冻结该实体的位移。

**0.18–0.68 秒：足部失力**

六足依次向身体下方内折，身体向一侧倾斜约 35 度，而不是现在的大幅侧翻。

毒囊脉冲速度加快两次，但亮度逐渐降低。

**0.68–1.08 秒：毒囊内爆**

毒囊向内部收缩，不做夸张爆炸。释放 5 枚小型、无伤害的装饰液滴，液滴落地形成几个暗色小点。

死亡时不生成有伤害的酸池，避免玩家因为击杀而受到惩罚。

**1.08–1.85 秒：残躯沉降**

尾部环抱身体，整个模型缩小高度并沉入一块暗色腐蚀痕迹。痕迹额外保留 `1.2 秒`，随后从外缘向中心收缩消失。

这套演出和它的技能语言统一：

```text
出生：毒液从地下聚合成生物
战斗：毒液被压入毒囊并投射
死亡：毒囊失压，生物重新溶回地面
```

它会比普通蜘蛛更加沉重、优雅，也能清楚表达它是专家怪。

## 实施顺序

建议按这个顺序落地：

1. 删除毒池奇偶扇区拼色，重做腐蚀液面。
2. 增加通用 `Despawning`，修复瞬间消失。
3. 将枪械改为 Hitscan 多目标穿透，并应用新伤害表。
4. 抽取统一 `PlanarCrowdSeparationSystem`，让所有怪物参与防堆叠。
5. 实现 Venom Lobber 的“毒囊裂土”和“毒囊塌缩”生命周期演出。

其中第一、二、三项会最快改善你当前截图中的视觉与战斗体验；第四项才是正确的底层架构修正。
