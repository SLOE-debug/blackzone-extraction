# 不规则程序化 Low Poly 美术规则

## 规则定位

本项目的统一三维画风定义为：**不规则、分面式、带洞穴岩体感的程序化 Low Poly**。

大厅的地面、墙面、天花板和双层祭台是该风格的基准：模型由代码直接生成低密度三角拓扑，通过非均匀轮廓、错落切面、确定性扰动和真实分面法线形成粗粝、拼接、岩石雕刻般的造型。

它可以称为 Low Poly，但项目要求不止是“面数较少”。规则同时要求：

1. 主体造型由程序生成；
2. 轮廓和表面具有受控的不规则性；
3. 三角分面能够参与视觉表达；
4. 模型具有明确的领域结构，而不是由通用 Primitive 堆叠而成。

## 适用范围

以下新增或重新设计的可见三维内容必须遵守本规则：

- 场景地面、墙壁、洞穴、建筑、台阶和门窗；
- 祭台、灯具、箱体、机械、家具和其他环境道具；
- 玩家、怪物、尸体、可破坏物和可拾取物；
- 具有实体表面的三维特效、碎片和装饰物。

以下内容不要求具有 Low Poly 造型，但不得作为最终可见模型混入画面：

- UI、调试面板、编辑器辅助线和性能诊断图形；
- 不可见碰撞体、导航代理和纯逻辑空间边界；
- 仅用于算法中间计算且不会进入渲染的几何数据。

现有模型不会仅因本规则自动重写；但后续只要对其主体视觉进行重做或扩展，就必须向本规则对齐。

## 视觉特征

### 低密度且有意图的拓扑

- 使用能够表达轮廓的最低合理分段数，不用高细分网格伪装 Low Poly。
- 三角形的方向、大小和疏密应服务于轮廓、转折、隆起、凹陷和受光关系。
- 大面积表面应具有可读的分面节奏，避免完全等距、完全同向的规则网格感。
- 固定拓扑优先在初始化前计算容量；动态实体也应保持稳定的拓扑契约。

### 非均匀轮廓

- 圆形应表现为低段数、不等半径或轻微错角的多边形轮廓。
- 方形应通过角点偏移、边长差异、局部外凸、内收或切角摆脱标准 Box 轮廓。
- 柱体、管道和肢体应允许截面半径、中心、旋转和分段长度发生受控变化。
- 左右两侧不应默认镜像得完全一致；只有领域语义明确要求对称时才保持对称。

### 分面与岩体感

- 墙面、地面、天花板和大型结构优先使用扰动 Grid、交替对角线、隆起、脊线和边缘衰减形成洞穴切面。
- 台阶、基座和祭台优先使用多圈轮廓、外凸肩部、内收层级和半径扰动形成雕凿感。
- 每个视觉平面应具有明确法线；需要硬切面的部位使用独立顶点和 Face Normal。
- 不得用平滑法线把本应可见的 Low Poly 转折全部抹平。

### 确定性不规则

- 不规则变化必须来自固定清单、类型化配置或带固定 seed 的确定性函数。
- 同一配置在每次启动、刷新和重新加载后必须生成相同造型。
- 禁止直接使用无种子随机数让静态模型每次运行发生变化。
- 扰动幅度必须受语义尺寸约束，不能让噪声破坏可行走面、碰撞边界或主体识别度。

## 强制建模方式

### 由项目代码生成最终 Mesh

- 主体可见轮廓必须由 TypeScript Geometry 模块计算 Position、Normal、Color、UV 和 Index 中实际需要的顶点流。
- Geometry 只负责拓扑、顶点、法线和语义区段，不得创建 Cocos Node、Material 或 MeshRenderer。
- Rendering 只负责 Cocos Mesh、材质、GPU 上传和节点生命周期，不得反向承载造型算法。
- 布局、轮廓、分段数、扰动幅度和 seed 应使用只读类型、枚举或类型化清单表达，不得散落魔法数字和字符串分发。
- 高频动态模型应遵守固定拓扑、TypedArray、SoA 和无逐帧临时对象的项目性能规则。

### 使用语义化程序几何

允许复用项目内稳定的数学与拓扑工具，但最终形体必须具有领域语义和风格化改造。例如：

- 扰动 Grid：洞穴墙面、地面、天花板和岩板；
- 多圈 Radial Shell：祭台、灯座、井口和环形机械；
- 不规则 Polyline Extrusion 或 Ribbon：裂缝、边框、梁和轨道；
- 低段数变截面 Tube：线缆、触手、肢体和管道；
- 不均匀 Triangle Fan：岩盘、灯口、封盖和碎片；
- 自定义低分段凸体：岩石、头部、躯干和不规则容器。

通用算法只能作为生成基础，不能未经领域变形就成为最终模型。

## 明确禁止

### 禁止 Cocos 内置 Primitive 拼装

- 禁止使用 Cocos 内置 Box、Sphere、Cylinder、Capsule、Cone、Plane、Quad、Torus 等 Primitive 作为最终可见模型。
- 禁止在编辑器中创建上述 Primitive，再通过缩放、旋转、穿插和节点堆叠拼成墙面、台阶、祭台、角色、怪物或道具。
- 禁止通过大量 Cube 充当砖块、台阶或岩壁，也禁止通过 Sphere 或 Capsule 拼接身体和关节。
- 禁止把内置 Primitive 隐藏在封装函数、Prefab 或通用工厂后绕过本规则。

Cocos 的 Camera、Light、MeshRenderer、Material、Mesh 和场景节点仍可正常使用；禁止的是内置通用几何体承担最终造型职责。

### 禁止规则原始几何冒充项目风格

- 禁止用自建的完美立方体、完美球体、等半径圆柱或高细分平面替代 Cocos Primitive 后声称符合规则。
- 禁止仅降低面数，却保留完全规则、光滑、机械复制般的轮廓。
- 禁止把多个互不融合的规则几何体简单穿插，形成明显的“积木拼装感”。
- 禁止为了省事使用单个平面贴图假装具有实际厚度和分面结构的主体模型。

### 禁止绕开程序化主体造型

- 除非用户明确指定外部美术资产，否则不得使用导入模型替代主体程序化造型。
- 图片、纹理和材质可以辅助表面表达，但不得代替应由几何体现的轮廓、层级、厚度和破面。
- 不得把复杂建模逻辑塞入场景入口、Bundle 入口或 Rendering 文件中。

## 典型对象的正确方向

| 对象 | 正确方向 | 禁止方向 |
| --- | --- | --- |
| 洞穴墙面 | 扰动网格、隆起、脊线、边缘闭合、交替三角切分 | 缩放 Plane 或排列 Cube |
| 台阶与祭台 | 多圈不等半径轮廓、外凸肩部、层级内收、固定扰动 | 叠放 Box 或 Cylinder |
| 岩石 | 低分段自定义凸体、方向性拉伸、不均匀半径 | 缩放 Sphere |
| 箱体与机械 | 偏移角点、切角、错层面板、非等距截面 | 完美 Box 拼装 |
| 灯具 | 低段数变截面壳体、不规则灯口和支架 | Cylinder、Cone 与 Sphere 穿插 |
| 角色与怪物 | 领域化低段数躯干、变截面肢体、受控不对称 | Sphere、Capsule 和 Cylinder 人偶 |

## 验收清单

新增可见三维模型完成前必须逐项确认：

1. 主体是否由项目代码生成，而非 Cocos Primitive 或默认模型？
2. 远看轮廓是否已经具有可识别的不规则性，而不是只在表面增加微小噪声？
3. 近看是否能读到有意图的三角分面、转折和层级？
4. 是否避免了完美 Box、Sphere、Cylinder 等通用几何轮廓？
5. 不规则参数是否确定、可复现并受尺寸约束？
6. Geometry、Rendering、布局配置和业务编排是否保持职责分离？
7. 法线、绕序、材质与真实灯光是否正确，而不是靠错误法线伪造明暗？
8. 模型是否与大厅墙面和双层祭台放在同一画面时仍属于同一美术语言？

任意一项不满足时，不得把模型视为符合项目最终画风。

## 风格基准

实现或评审新模型时，优先参考以下现有代码：

- `assets/lobby/geometry/recipes/lobby-shell-recipe.ts`：大厅各曲面的类型化配方；
- `assets/lobby/geometry/deformers/lobby-surface-deformer.ts`：确定性洞穴隆起与切向扰动；
- `assets/lobby/geometry/lobby-altar-geometry.ts`：双层不规则多圈祭台；
- `assets/lobby/geometry/lobby-altar-layout.ts`：固定层级、低段数和半径扰动；
- `assets/lobby/geometry/lobby-geometry-topology.ts`：固定拓扑与独立分面顶点契约。

风格判断以最终轮廓、分面结构和程序化生成方式为准，不以文件名中是否包含 `low-poly` 为准。
